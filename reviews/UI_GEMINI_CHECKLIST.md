# Gemini UI Review Checklist — feature/ui-001-institutional-theme

Purpose: verifier (Gemini) performs a non-runtime UI/UX validation and documents precise acceptance criteria. No app build or run required.

## A. 색 대비 수치 점검 (Contrast / WCAG)

검사 대상 파일: `src/app/globals.css`

- 측정 항목:
   - `--bg` (배경) vs `--text` (본문 텍스트)
   - `--panel` vs `--text` (카드/패널 텍스트)
   - `--panel2` vs `--text`
   - `--kill`/`--warn` 사용시 강조 텍스트(버튼·배지) 대비

- 요구 수치 (우선순위):
   - 본문 텍스트/배경: 최소 WCAG AA (대비비 >= 4.5:1)
   - 큰 텍스트(≥18pt 또는 bold ≥14pt): WCAG AA (>= 3:1)
   - UI 컴포넌트(버튼·배지)의 색상 대비는 상호작용성 고려, 상태 변별이 명확해야 함

- 수행 방법 (수동/도구):
   - 색상 코드로 대비비 계산(예: https://contrast-ratio.com 또는 `axe` 색상 체크)
   - 결과 표로 문서화: 컬러 페어, 대비비, 판정(통과/경고/불가)

## B. RiskBar 상태 UX 기준 (명문화)

모드별 시각 기준(강제 준수 사항):

| Mode | 시각 기준 |
|---|---|
| NORMAL | 기본 border 사용 (기본 상태, 배경 변화 없음) |
| WARN | 상단에 얇은 앰버(오렌지) 라인 또는 배지로 경고 표시. 버튼은 활성 상태 유지. |
| KILL | 상단 강조(두꺼운 빨간 라인) + 주요 조작 버튼은 disabled 처리(비활성화 스타일 명시). 색상은 `--kill` 사용. |
| DOWNGRADE | 제한 모드 텍스트(예: "Downgrade — 일부 기능 제한")를 명시적으로 표시. 배경 전체 색 변경은 금지. |

추가 규칙:
- 상태 전환 시 페이지 전체 배경/테마 변경 금지(기관형 절제 유지).
- `KILL` 모드 진입 전 사용자 확인 흐름(애니메이션/경고 팝업)은 디자인 상 표시 권장 — 단, 이 PR에서는 UI 토글만 구현.

## C. Layout 안정성 검증 항목

- 기본 가정 해상도: 디자인 기준은 1440px 너비에서 주요 레이아웃(좌측 네비게이션 260px, 본문 2:1 그리드)이 정상 동작해야 함.
- 최소 지원 해상도 권장: 1280px width (노트북), 1024px는 축소/레이아웃 변경(사이드바 숨김) 전략 필요.
- 반응형 브레이크포인트 점검:
   - >=1440px: 데스크톱 레이아웃(사이드바 고정)
   - 1024–1439px: 사이드바 collapsible 또는 overlay 권장
   - <1024px: 모바일/태블릿 전용 레이아웃 필요(본 PR 범위 외)

- 체크포인트 항목:
   - 그리드(2fr 1fr)가 1440px에서 두 영역을 의도대로 배치하는지 확인
   - 카드 내부 패딩/여백이 줄어들어도 숫자 가독성이 유지되는지 확인

## D. 투자자 데모 관점 체크

- 한 화면에서 즉시 보이는 항목(우선순위):
   1. 리스크 모드(상단 오른쪽, GlobalRiskBar)
   2. 메인 마켓/차트(왼쪽 큰 패널)
   3. 오더북/포지션(오른쪽 패널)

- 데모 관점 평가 항목:
   - "리스크 통제 시스템"이 한눈에 느껴지는가 (RiskBar 가시성, 상태 레이블)
   - 네온/게임 UI 느낌(강한 발광·그라데이션) 금지 — 기관형 절제 유지
   - 숫자(가격·포지션) 폰트 크기/가독성 우선: 최소 14px 권장, 핵심 숫자는 16–18px

- 상태별 스냅샷(모드 NORMAL/WARN/KILL/DOWNGRADE) — 정적 캡처 이미지 권장
- 반응형 체크 메모(1440px/1280px/1024px)
- 최종 검토 코멘트 및 승인(이 문서에 타임스탬프 기재)
